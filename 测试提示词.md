# Hermes Agent MCP Space - 全面功能测试提示词

> 将以下提示词发送给任意智能体（Claude/GPT/Gemini 等），让它通过 MCP 协议自动测试所有功能。

---

## 提示词正文

```
你是一个专业的 QA 测试工程师。现在请通过 MCP 协议对 Hermes Agent 进行全面功能测试。

### 测试环境
- MCP 端点: https://arwei944-hermes-mcp-space.hf.space/mcp
- 协议: MCP Streamable HTTP (JSON-RPC 2.0)
- 调用方式: POST /mcp，Content-Type: application/json

### 调用格式
每次调用发送以下格式的请求：
```json
{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{"name":"工具名","arguments":{参数}}}
```

### 测试要求
1. 按顺序测试以下所有功能模块
2. 每个测试步骤都要实际调用 MCP 工具
3. 记录每个测试的结果（成功/失败/返回内容摘要）
4. 如果某个测试失败，记录错误信息并继续下一个
5. 最后输出完整的测试报告

---

## 第一步：系统状态检查

### 1.1 获取系统状态
调用 `get_system_status`，验证返回包含 version、uptime 等字段。

### 1.2 获取仪表盘摘要
调用 `get_dashboard_summary`，验证返回统计数据。

### 1.3 获取操作日志
调用 `get_logs`，验证返回日志列表。

---

## 第二步：会话管理（CRUD）

### 2.1 创建会话
调用 `create_session`，参数：
- title: "QA 自动测试会话"
- source: "qa_test"
记录返回的 session_id，后续步骤会用到。

### 2.2 列出所有会话
调用 `list_sessions`，验证刚创建的会话出现在列表中。

### 2.3 记录对话消息
调用 `log_conversation`，参数：
- role: "user"
- content: "这是一条 QA 自动测试消息，用于验证对话记录功能"
- session_id: 使用 2.1 步骤创建的 session_id

### 2.4 记录助手回复
调用 `log_conversation`，参数：
- role: "assistant"
- content: "QA 测试回复：对话记录功能正常"
- session_id: 同上

### 2.5 记录带摘要的消息
调用 `log_conversation`，参数：
- role: "user"
- content: "测试摘要功能"
- summary: "QA 摘要测试"
- session_id: 同上

---

## 第三步：技能管理（CRUD）

### 3.1 列出所有技能
调用 `list_skills`，记录当前技能数量。

### 3.2 创建技能
调用 `create_skill`，参数：
- name: "qa-test-skill"
- content: "# QA 测试技能\n\n这是一个自动化测试创建的技能。\n\n## 功能\n- 测试技能创建\n- 测试技能删除"
- description: "QA 自动测试用技能"
- tags: ["test", "qa"]

### 3.3 更新技能
调用 `update_skill`，参数：
- name: "qa-test-skill"
- content: "# QA 测试技能（已更新）\n\n内容已更新。"

### 3.4 再次列出技能
调用 `list_skills`，验证 qa-test-skill 出现在列表中。

### 3.5 删除技能
调用 `delete_skill`，参数：
- name: "qa-test-skill"

### 3.6 验证删除
调用 `list_skills`，验证 qa-test-skill 已不在列表中。

---

## 第四步：记忆管理

### 4.1 读取当前记忆
调用 `read_memory`，记录当前记忆内容。

### 4.2 写入记忆
调用 `write_memory`，参数：
- content: "QA 测试记忆：这条记忆由自动化测试写入于 2026-04-28"

### 4.3 再次读取记忆
调用 `read_memory`，验证新写入的记忆内容存在。

---

## 第五步：用户画像

### 5.1 读取用户画像
调用 `read_user_profile`，记录返回内容。

### 5.2 更新用户画像
调用 `write_user_profile`，参数：
- content: "## 用户画像\n- 角色: QA 测试工程师\n- 偏好: 自动化测试\n- 测试日期: 2026-04-28"

### 5.3 验证更新
调用 `read_user_profile`，验证画像已更新。

---

## 第六步：定时任务

### 6.1 列出定时任务
调用 `list_cron_jobs`，记录当前任务数量。

### 6.2 创建定时任务
调用 `create_cron_job`，参数：
- name: "qa-test-job"
- cron_expression: "0 0 31 12 *"（每年 12 月 31 日执行，不会真正触发）
- action: "test"
- target: "qa"

### 6.3 验证创建
调用 `list_cron_jobs`，验证 qa-test-job 出现。

### 6.4 删除定时任务
调用 `delete_cron_job`，参数：
- job_id: 使用 6.2 创建的任务 ID（从 list_cron_jobs 返回中获取）

---

## 第七步：插件系统

### 7.1 列出已安装插件
调用 `list_plugins`，记录当前插件数量。

### 7.2 列出可用插件工具
调用 `list_tools`，记录工具总数（应 ≥ 28）。

### 7.3 安装内置插件
调用 `install_plugin`，参数：
- name: "code-analyzer"

### 7.4 验证安装
调用 `list_plugins`，验证 code-analyzer 已安装。

### 7.5 卸载插件
调用 `uninstall_plugin`，参数：
- name: "code-analyzer"

### 7.6 验证卸载
调用 `list_plugins`，验证 code-analyzer 已卸载。

---

## 第八步：配置管理

### 8.1 获取当前配置
调用 `get_config`，记录配置内容。

### 8.2 更新配置
调用 `update_config`，参数：
- key: "qa_test_flag"
- value: "tested_at_2026-04-28"

### 8.3 验证更新
调用 `get_config`，验证 qa_test_flag 存在。

---

## 第九步：工具列表验证

### 9.1 列出所有工具
调用 `list_tools`，验证以下 28 个工具全部存在：

**会话类 (4)**
- list_sessions, create_session, search_sessions, get_session_messages

**技能类 (4)**
- list_skills, get_skill_content, create_skill, update_skill, delete_skill

**记忆类 (4)**
- read_memory, write_memory, read_user_profile, write_user_profile

**定时任务 (3)**
- list_cron_jobs, create_cron_job, delete_cron_job

**系统类 (4)**
- get_system_status, get_dashboard_summary, get_logs, get_config, update_config

**插件类 (3)**
- list_plugins, install_plugin, uninstall_plugin, list_tools

**对话类 (1)**
- log_conversation

---

### 10.1 获取不存在的技能
调用 `get_skill_content`，参数：
- name: "nonexistent-skill-xyz"
预期：返回错误信息，不应崩溃。

### 10.2 删除不存在的技能
调用 `delete_skill`，参数：
- name: "nonexistent-skill-xyz"
预期：返回错误信息，不应崩溃。

### 10.3 获取不存在的会话消息
调用 `get_session_messages`，参数：
- session_id: "nonexistent-session-xyz"
预期：返回错误信息或空列表，不应崩溃。
